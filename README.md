# IS-Student-AI-Assistant

Локальный помощник для студентов информационных систем: работа с SQL, проектирование ER-диаграмм, нормализация баз данных и хранение истории проектных решений.

> Коротко: это инструмент на Python, который помогает изучать и практиковаться в предметах информационных систем — генерировать запросы SQL, проверять/улучшать ER-диаграммы, выполнять шаги нормализации и сохранять историю работы над проектом для ревью и воспроизведения результатов.

## Возможности
- Генерация и проверка SQL-запросов (SELECT/INSERT/UPDATE/DELETE, JOIN, агрегатные функции).
- Создание и визуальная проверка ER-диаграмм (сгенерированные схемы можно экспортировать).
- Поддержка пошаговой нормализации (1NF → 2NF → 3NF и рекомендации по денормализации).
- Локальный журнал/история проекта: сохранение версий таблиц, диаграмм и заметок по решению задач.
- Интерактивные подсказки и рекомендации (подсказки по индексации, связям, целостности данных).
- Поддержка шаблонов заданий/шаблонов баз данных для учебных задач.

## Технологии

(Точные зависимости смотрите в файле `requirements.txt` репозитория.)

## Установка
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Baxa2367/IS-Student-AI-Assistant.git
   cd IS-Student-AI-Assistant
   ```

2. Создайте виртуальное окружение и активируйте его:
   - Linux/macOS:
     ```bash
     python3 -m venv venv
     source venv/bin/activate
     ```
   - Windows:
     ```powershell
     python -m venv venv
     .\venv\Scripts\Activate.ps1
     ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. (Опционально) Установите Graphviz для экспорта диаграмм:
   - macOS (Homebrew):
     ```bash
     brew install graphviz
     ```
   - Ubuntu/Debian:
     ```bash
     sudo apt-get install graphviz
     ```
   - Windows: скачайте установщик с официального сайта Graphviz и добавьте путь в PATH.

## Примеры использования

1) Проверка и генерация SQL-запроса:
- Задача: "Напиши запрос, который выбирает имена студентов и средний балл по предмету 'Базы данных', сгруппированный по студенту, только для оценок выше 3."
- CLI:
  ```bash
  assistant sql "students, grades, courses" "SELECT name, AVG(score) FROM ... WHERE course = 'Базы данных' AND score > 3 GROUP BY name"
  ```
- Результат: сгенерированный корректный SQL, и рекомендации по индексам.

2) Нормализация таблицы:
- Ввод: таблица с полями (order_id, customer_name, customer_address, product_id, product_name, quantity)
- Вывод: шаги нормализации с разбивкой на таблицы orders, customers, products, order_items и объяснение нарушений 1NF/2NF/3NF и предложенных ключей.

3) ER-диаграмма:
- Импорт схемы или ввод сущностей → генерация визуального представления (PNG/SVG) или вывод в формате DOT.

4) История проекта:
- Сохранение версии структуры БД, диаграммы и комментария: `assistant save --name "v1 initial design" --notes "Добавлена таблица orders"`

(Примеры интерфейса — ориентир. Конкретные команды могут отличаться. Проверьте реальные CLI-параметры в коде.)

## Архитектура и модули (рекомендуемая структура)
Примерная структура проекта:
```
IS-Student-AI-Assistant/
├── assistant/             # основной Python пакет
│   ├── cli.py             # команды CLI
│   ├── web.py             # (опционально) web-интерфейс
│   ├── sql_tools.py       # генерация и валидация SQL
│   ├── er_generator.py    # генерация ER-диаграмм (Graphviz/NetworkX)
│   ├── normalization.py   # логика нормализации
│   ├── history.py         # управление историей/версиями проекта
│   └── utils.py
├��─ tests/                 # автотесты
├── requirements.txt
├── README.md
└── examples/              # примеры схем, SQL и т.д.
```

Ключевые модули:
- sql_tools.py — формирование SQL, анализ планов (если подключён локальный СУБД), проверка синтаксиса.
- er_generator.py — перевод схемы в визуальную форму, экспорт в DOT/PNG/SVG.
- normalization.py — функции для проверки нормальных форм и предложений по рефакторингу схемы.
- history.py — локальное хранилище версий (JSON/SQLite) для отслеживания прогресса.

## Конфиденциальность и локальная работа
Проект рассчитан на локальную работу — данные пользователей (схемы, БД, задания) остаются на машине. Если вы добавляете интеграции с удалёнными ML-сервисами или API, добавляйте явную опцию/настройки и детально документируйте требования по API-ключам и приватности.

Рекомендации:
- Не коммите реальные базы данных или чувствительные данные в репозиторий.
- Для истории используйте локальный файл (history.json) или локальную SQLite DB, зашифровывайте при необходимости.

## Частые вопросы (FAQ)
- Нужно ли подключать интернет для работы?
  - Нет, базовая функциональность — локальная. Однако некоторые опции (если добавлены) для NLP/ML могут требовать внешних сервисов.
- Как импортировать существующую структуру БД?
  - Импорт возможен в формате SQL-дампа или в виде CSV-файлов (см. `examples/`).
- Поддерживаются ли конкретные СУБД?
  - Для проверки синтаксиса можно использовать SQLAlchemy-парсеры; специфические диалекты (Postgres, MySQL) требуют дополнительных адаптеров.
